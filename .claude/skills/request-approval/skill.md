# request-approval

Create a human-in-the-loop approval request file for sensitive actions.

## Trigger

Use this skill when about to take ANY of these actions:
- Send an email to a new/unknown contact
- Send any email (reply or new)
- Make or schedule any payment
- Post on social media (LinkedIn, Twitter, Instagram, Facebook)
- Delete any file
- Contact any external party on behalf of the business
- Any action involving money > $50

NEVER act directly on these — always create an approval file first.

## Instructions

1. **Determine the action details**
   - What exactly needs to happen?
   - Who is the recipient/target?
   - What is the content/amount?
   - Is this reversible?

2. **Create the approval file**
   - Filename: `/Pending_Approval/<TYPE>_<recipient>_<YYYYMMDD_HHMMSS>.md`
   - Types: `EMAIL`, `PAYMENT`, `SOCIAL_POST`, `DELETE`, `EXTERNAL_ACTION`
   - Use the template below

3. **Stop and wait**
   - Do NOT execute the action
   - Update the parent plan (if any) noting approval is pending
   - Update Dashboard.md: increment Pending Approvals count

4. **Check /Approved/ folder**
   - The orchestrator watches /Approved/ and re-triggers Claude
   - When the file appears in /Approved/, execute the action
   - After execution: move to /Done/, log the action

## Approval File Template

```markdown
---
type: approval_request
action: <email_send|payment|social_post|file_delete|external_action>
priority: <high|medium|low>
recipient: <who this affects>
amount: <if financial, e.g. $500.00>
created: <ISO timestamp>
expires: <ISO timestamp — 24 hours from created>
status: pending
source_plan: <Plans/PLAN_xxx.md if part of a plan>
---

## Action Requested

**Type**: <action type>
**To / Target**: <recipient or target>
**Subject / Reference**: <brief description>

## Details

<Full details of what will happen if approved>

## Reason

<Why this action is needed — context from the triggering item>

## Risk Assessment

- **Reversible**: Yes / No
- **Financial impact**: $<amount> / None
- **External parties affected**: <list>

## ✅ To Approve

Move this file to `/Approved/` folder.

## ❌ To Reject

Move this file to `/Rejected/` folder.

---
*Auto-generated by AI Employee | Expires: <expiry timestamp>*
```

## Security Rules

- NEVER auto-approve a payment to a new payee
- NEVER auto-approve payments > $100
- NEVER send emails to unknown contacts without approval
- If approval file expires (>24h), move to /Rejected/ and create new request
