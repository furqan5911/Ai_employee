## 🔑 Key Technical Decisions Made

1. **DRY_RUN=true by default** — safe for testing, set to false when ready for real actions
2. **`base_watcher.py` as abstract class** — all watchers inherit the polling loop, making Gmail/WhatsApp watchers consistent
3. **JSON Lines format for logs** — one JSON object per line, easy to parse and append
4. **Claim-by-move rule** — task moved to `/In_Progress/` to prevent duplicate processing
5. **Ralph Wiggum uses exit code 2** — Claude Code's rewake signal to re-inject prompt
6. **Promise-based + file-move completion** — two strategies, both implemented in ralph_wiggum.py
7. **Agent Skills for all AI functionality** — per docs requirement, not just raw prompts
